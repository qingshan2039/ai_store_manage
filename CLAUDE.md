# CLAUDE.md

本项目(AI 门店管理系统)的开发流程铁律,所有协作必须严格遵守以下 6 条。

阶段与目录对应关系:

- **Contract(契约)** → `api-contract/`(openapi.yaml)
- **Backend(后端)** → `api/`(Spring Boot)
- **Frontend(前端)** → `web/`(React)

## 1. 先 Contract,后 Backend,再 Frontend

严格按 `api-contract/` → `api/` → `web/` 的顺序推进,不得跳阶段、不得并行抢跑。先定义接口契约,再实现后端,最后对接前端。

## 2. 未验证,不进入下一阶段

当前阶段未通过验证(可编译、可运行、接口/页面实际可用)之前,不得进入下一阶段。

## 3. 未定义,不允许实现

任何未在契约(openapi.yaml)中定义的接口、字段、行为,一律不得实现。契约是唯一依据;如有需要,先改契约,再实现。

## 4. 只改当前阶段目录,禁止跨目录修改

Contract 阶段只改 `api-contract/`,Backend 阶段只改 `api/`,Frontend 阶段只改 `web/`。禁止在某一阶段修改其他阶段的目录。

## 5. 每次输出必须完整、可运行、可验证

交付内容必须完整、可直接运行,并给出验证方式(命令 / curl / 测试步骤)。禁止输出片段、伪代码或"待补全"内容。

## 6. 返回必须是中文

所有回复与说明一律使用中文。
